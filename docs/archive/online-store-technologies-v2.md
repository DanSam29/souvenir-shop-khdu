# Технології інтернет-магазину  
1. Frontend - відповідає за відображення сайту, зручність користування, швидкість завантаження сторінок.  
   Варіанти:  
     - React  
     - Vue.js  
     - Angular  

   Бажаний вибір: React (показує чітке розділення між frontend та backend, дозволяє продемонструвати REST API, а також дає практику роботи з асинхронними викликами, станом клієнта і авторизацією).  
2. Backend - обробляє запити з фронтенду, керує бізнес-логікою, перевіряє права доступу.  
   Варіанти:  
     - Python + Django (+ Django REST Framework)  
     - Node.js + Express або NestJS (TypeScript)  
     - PHP + Laravel  
     - Java + Spring Boot  
     - .NET Core (C#)  

   Бажаний вибір: .NET Core (C#) (фреймворк для створення RESTful API, який чудово інтегрується з React і SQL-базами).  
3. БД - зберігає інформацію про товари, користувачів, замовлення, фінанси.  
   Варіанти:  
     - PostgreSQL  
     - MySQL / MariaDB  
     - SQLite  
     - MongoDB  

   Бажаний вибір: MySQL (популярна реляційна база даних, сумісна з .NET Core через Entity Framework Core).  
4. ORM (рівень доступу до даних) - забезпечує взаємодію бекенду з базою даних, дозволяє створювати, читати, оновлювати і видаляти дані.  
   Варіанти:  
     - Django ORM  
     - SQLAlchemy (Python)  
     - Prisma (для Node.js)  
     - Eloquent (Laravel)  
     - Entity Framework Core  

   Бажаний вибір: Entity Framework Core (зручний інструмент .NET для роботи з MySQL через моделі C#, підтримує міграції та транзакції).  
5. Cache (кешування та черги фонових задач) - прискорює роботу сайту та обробку асинхронних завдань, наприклад, відправку повідомлень або оновлення статусів.  
   Варіанти:  
     - Redis (cache + broker) ($5/міс)  
     - RabbitMQ (broker)  
     - Celery (task runner, у парі з Redis/RabbitMQ)  
     - Sidekiq / Bull (Node.js)  
     - In-Memory Cache  

   Бажаний вибір: In-Memory Cache (вбудований в .NET, простий для курсового проекту, не потребує додаткового сервера).  
6. Пошук по каталогу / фільтрація - реалізує швидкий пошук та сортування товарів для зручності користувача.  
   Варіанти:  
     - Postgres full-text search  
     - MySQL full-text search  
     - Elasticsearch / OpenSearch ($20–$30/міс)  
     - Algolia (SaaS)  
     - Meilisearch  

   Бажаний вибір: MySQL full-text search (не потребує додаткових сервісів; дозволяє реалізувати пошук та сортування товарів безкоштовно).  
7. Аутентифікація / Авторизація - керує входом користувачів, ролями і правами доступу до різних частин системи.  
   Варіанти:  
     - JWT (JSON Web Tokens)  
     - ASP.NET Identity  
     - Session cookies (Django sessions)  
     - OAuth2 / OpenID Connect (Google, Facebook)  
     - RBAC / ACL (рівні дозволів)  

   Бажаний вибір: ASP.NET Identity + JWT для API (вбудовані інструменти .NET Core для безпечного входу та ролей користувачів, підтримує токени для React API).  
8. Платежі (Fіnancial flow) - обробляє транзакції, інтегрується з платіжними шлюзами і веде облік оплати замовлень.  
   Варіанти:  
     - Fondy (2.5% комісія з транзакцій)  
     - WayForPay (2.5% комісія з транзакцій)  
     - LiqPay (2.75% комісія з транзакцій)  
     - Stripe (2.9% + $0.30 за транзакцію)  
     - PayPal (3.4% + $0.30 за транзакцію)  

   Бажаний вибір: Stripe sandbox (безкоштовний тестовий режим для демонстрації логіки платежів без реальних транзакцій).  
9. Облік товарів і фінансовий облік - відстежує залишки на складі, створює записи фінансових операцій і формує звіти.  
   Варіанти:  
     - Реалізація в реляційній БД (таблиці Stock, Transactions, Payments)  
     - Event Sourcing / Audit log (для фінансового відстеження)  

   Бажаний вибір: MySQL таблиці + Аudit log (забезпечує просту реалізацію транзакцій, фінансовий облік і можливість перевірки змін).  
10. Зберігання медіа-файлів (зображення товарів) - зберігає медіа-файли товарів та забезпечує їх доступність для фронтенду.  
    Варіанти:  
      - Локальний filesystem  
      - S3-сумісне сховище: AWS S3 ($0.023/ГБ), DigitalOcean Spaces ($5/міс), MinIO (локально)  
      - CDN (Cloudflare, CloudFront)  

    Бажаний вибір: MinIO (локально для розробки) (MinIO дозволяє емулювати S3 локально (простий для демонстрації)).  
11. API документація - описує ендпоінти і методи взаємодії для фронтенду та зовнішніх інтеграцій.  
    Варіанти:  
      - OpenAPI / Swagger (drf-spectacular / drf-yasg для Django)  
      - Postman Collection  
      - GraphQL Playground (якщо GraphQL)  

    Бажаний вибір: Swagger (автоматична генерація документації для .NET API, зручно для тестування та демонстрації).  
12. Тестування - перевіряє коректність роботи бекенду і фронтенду, покращує стабільність сайту.  
    Варіанти:  
      - pytest (Python)  
      - unittest (Python stdlib)  
      - Jest (frontend)  
      - xUnit/NUnit (backend)  
      - Cypress / Selenium (end-to-end)  

    Бажаний вибір: xUnit + Jest + Cypress (xUnit для тестування .NET логіки, Jest для React компонентів, Cypress для 1–2 E2E сценаріїв).  
13. Контейнеризація та локальний дев - забезпечує відтворюване середовище для розробки і демонстрації проєкту.  
    Варіанти:  
      - Docker + Docker Compose  
      - LXC / Vagrant / VM  
      - Безконтейнерний запуск локально  

    Бажаний вибір: Docker + Docker Compose (дозволяє локально запускати всі сервіси і демонструвати проєкт у відтворюваному середовищі).  
14. CI / CD - автоматизує процес збірки, тестування і розгортання проєкту.  
    Варіанти:  
      - GitHub Actions  
      - GitLab CI  
      - Jenkins / Drone / CircleCI  

    Бажаний вибір: GitHub Actions (простота налаштування для студентського репо, інтеграція з Docker, CI pipeline (lint → test → build)).  
15. Хостинг - забезпечує доступність сайту в інтернеті та надає серверні ресурси.  
    Варіанти:  
      - DigitalOcean (Droplet / App Platform) ($4–$12/міс)  
      - Heroku ($5–$10/міс) / Render (PaaS)  
      - AWS (EC2, RDS, S3)  
      - VDS / VPS  
      - Render.com  

    Бажаний вибір: Render.com (підтримує .NET Core, безкоштовний тариф для навчальних проектів).  
16. Логування та моніторинг - відстежує помилки, збирає статистику роботи системи та допомагає в аналізі продуктивності.  
    Варінати:  
      - Sentry (error tracking)  
      - Prometheus + Grafana (метрики)  
      - CloudWatch / Papertrail (логування)  
      - Serilog  

    Бажаний вибір: Serilog + локальне логування (збирає логи .NET Core, безкоштовне для курсового проєкту).  
17. Панель управління - надає адміністраторам інтерфейс для керування товарами, замовленнями, користувачами і фінансами.  
    Варіанти:  
      - Вбудована Django Admin  
      - Вбудована React Admin  
      - Панель управління на React (custom dashboard)  
      - Використання готових UI kit / shadcn, AdminLTE  

    Бажаний вибір: React Admin (легко інтегрується з REST API .NET і швидко дозволяє створити зручну адмінку на React).  
18. Логістика - інтегрує зовнішні сервіси доставки, створює накладні та відстежує статус посилок.  
    Варіанти:  
      - Інтеграції з локальними API (Nova Poshta, Ukrposhta)  
      - Інтеграція з міжнародними перевізниками (DHL, UPS)  
      - Модуль manual-entry (ручне оформлення)  

    Бажаний вибір: Nova Poshta API (демонстраційна локальна інтеграція для України; доступна документація та sandbox).  
19. Безпека - забезпечує захист даних і транзакцій, реалізує протоколи HTTPS, CSRF, XSS, перевірку вводу та управління секретами.  
    Варіанти:  
      - HTTPS (Let's Encrypt)  
      - CSRF, XSS-захист (фреймворк)  
      - Rate-limiting, input validation, secrets manager  

    Бажаний вибір: HTTPS + CSRF + валідовані ендпоінти + секрети в env (базові, але обов’язкові практики для захисту даних і платежів; демонструють знання безпеки у проєкті).
## Визначена конфігурація:
1. Backend: React  
2. Frontend: .NET Core (C#)  
3. БД: MySQL  
4. ORM: Entity Framework Core  
5. Cache: In-Memory Cache  
6. Пошук по каталогу / фільтрація: MySQL full-text search  
7. Аутентифікація / Авторизація: ASP.NET Identity + JWT для API  
8. Платежі: Stripe sandbox  
9. Облік товарів і фінансовий облік: MySQL таблиці + Аudit log  
10. Зберігання медіа-файлів: MinIO (локально для розробки)  
11. API документація: Swagger  
12. Тестування: xUnit + Jest + Cypress  
13. Контейнеризація та локальний дев: Docker + Docker Compose  
14. CI / CD: GitHub Actions  
15. Хостинг: Render.com  
16. Логування та моніторинг: Serilog + локальне логування  
17. Панель управління: React Admin  
18. Логістика: Nova Poshta API  
19. Безпека: HTTPS + CSRF + валідовані ендпоінти + секрети в env
