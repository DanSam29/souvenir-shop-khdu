# Технології інтернет-магазину
1. Frontend - відповідає за відображення сайту, зручність користування, швидкість завантаження сторінок.  
   Варіанти:  
     - React  
     - Vue.js  
     - Angular  

   Бажаний вибір: React (показує чітке розділення між frontend та backend, дозволяє продемонструвати REST API, а також дає практику роботи з асинхронними викликами, станом клієнта і авторизацією).  
2. Backend - обробляє запити з фронтенду, керує бізнес-логікою, перевіряє права доступу.  
   Варіанти:  
     - Python + Django (+ Django REST Framework)  
     - Node.js + Express або NestJS (TypeScript)  
     - PHP + Laravel  
     - Java + Spring Boot  
     - .NET Core (C#)  

   Бажаний вибір: Python + Django (швидка розробка, вбудований ORM та панель управління, дає зручну реалізацію REST API; дуже широка документація, що мінімізує витрати на реалізацію).  
3. БД - зберігає інформацію про товари, користувачів, замовлення, фінанси.  
   Варіанти:  
     - PostgreSQL  
     - MySQL / MariaDB  
     - SQLite  
     - MongoDB  

   Бажаний вибір: PostgreSQL (добре інтегрується з Django та підходить для фінансового обліку та резервування товарів).  
4. ORM (рівень доступу до даних) - забезпечує взаємодію бекенду з базою даних, дозволяє створювати, читати, оновлювати і видаляти дані.  
   Варіанти:  
     - Django ORM  
     - SQLAlchemy (Python)  
     - Prisma (для Node.js)  
     - Eloquent (Laravel)  

   Бажаний вибір: Django ORM (природна частина Django, прості міграції, зв’язки, транзакції).  
5. Cache (кешування та черги фонових задач) - прискорює роботу сайту та обробку асинхронних завдань, наприклад, відправку повідомлень або оновлення статусів.  
   Варіанти:  
     - Redis (cache + broker)  
     - RabbitMQ (broker)  
     - Celery (task runner, у парі з Redis/RabbitMQ)  
     - Sidekiq / Bull (Node.js)  

   Бажаний вибір: Redis + Celery (Redis простий у розгортанні; Celery - стандарт для фонових задач у Python (відправлення емейлів, оновлення статусів замовлень, синхронізація з зовнішніми API)).  
6. Пошук по каталогу / фільтрація - реалізує швидкий пошук та сортування товарів для зручності користувача.  
   Варіанти:  
     - Postgres full-text search  
     - Elasticsearch / OpenSearch  
     - Algolia (SaaS)  
     - Meilisearch  

   Бажаний вибір: Postgres full-text search (не потребує додаткового сервісу; дозволяє реалізувати пошук і фільтри на основі наявної БД).  
7. Аутентифікація / Авторизація - керує входом користувачів, ролями і правами доступу до різних частин системи.  
   Варіанти:  
     - JWT (JSON Web Tokens)  
     - Session cookies (Django sessions)  
     - OAuth2 / OpenID Connect (Google, Facebook)  
     - RBAC / ACL (рівні дозволів)  

   Бажаний вибір: JWT для API (клієнт React) + Django sessions для панелі управління (комбінація дозволяє показати сучасний API-підхід і безпечну серверну панель управління на сесіях).  
8. Платежі (Fіnancial flow) - обробляє транзакції, інтегрується з платіжними шлюзами і веде облік оплати замовлень.  
   Варіанти:  
     - Fondy  
     - WayForPay  
     - LiqPay  
     - Stripe  
     - PayPal  

   Бажаний вибір: Fondy (локальний шлюз) + тестовий режим Stripe (Fondy - реалістичний український варіант з документацією та прикладами; Stripe - демонструє інтернаціональні підходи, дає змогу використовувати sandbox, щоб не робити реальних транзакцій під час розробки).  
9. Облік товарів і фінансовий облік - відстежує залишки на складі, створює записи фінансових операцій і формує звіти.  
   Варіанти:  
     - Реалізація в реляційній БД (таблиці Stock, Transactions, Payments)  
     - Event Sourcing / Audit log (для фінансового відстеження)  

   Бажаний вибір: Реляційна модель у PostgreSQL (таблиці Orders, OrderItems, Stock, Payments, FinancialTransactions) + audit log (логування операцій у окрему таблицю) (забезпечує можливість відкату (транзакції), коректний фінансовий облік і простоту доведення коректності в звітах; Audit log дає слід операцій для перевірок).  
10. Зберігання файлів (зображення товарів) - зберігає медіа-файли товарів та забезпечує їх доступність для фронтенду.  
    Варіанти:  
      - Локальний filesystem  
      - S3-сумісне сховище: AWS S3, DigitalOcean Spaces, MinIO (локально)  
      - CDN (Cloudflare, CloudFront)  

    Бажаний вибір: MinIO (локально для розробки) + DigitalOcean Spaces / S3 для продакшену (MinIO дозволяє емулювати S3 локально (простий для демонстрації); S3/Spaces - продакшн-підхід з CDN можливістю).  
11. API документація - описує ендпоінти і методи взаємодії для фронтенду та зовнішніх інтеграцій.  
    Варіанти:  
      - OpenAPI / Swagger (drf-spectacular / drf-yasg для Django)  
      - Postman Collection  
      - GraphQL Playground (якщо GraphQL)  

    Бажаний вибір: OpenAPI/Swagger (drf-spectacular) (автоматична генерація документів для DRF; зручно для демонстрації ендпоінтів, тестування й оцінки API).  
12. Тестування - перевіряє коректність роботи бекенду і фронтенду, покращує стабільність сайту.  
    Варіанти:  
      - pytest (Python)  
      - unittest (Python stdlib)  
      - Jest (frontend)  
      - Cypress / Selenium (end-to-end)  

    Бажаний вибір: pytest (backend) + Jest (frontend unit) + Cypress (1–2 E2E тести) (pytest - зручний, багато плагінів; Jest - стандарт для React; Cypress дає наочну автоматизовану перевірку сценаріїв (checkout flow)).  
13. Контейнеризація та локальний дев - забезпечує відтворюване середовище для розробки і демонстрації проєкту.  
    Варіанти:  
      - Docker + Docker Compose  
      - LXC / Vagrant / VM  
      - Безконтейнерний запуск локально  

    Бажаний вибір: Docker + Docker Compose (відтворюване середовище, легкий деплой, інтеграція в CI).  
14. CI / CD - автоматизує процес збірки, тестування і розгортання проєкту.  
    Варіанти:  
      - GitHub Actions  
      - GitLab CI  
      - Jenkins / Drone / CircleCI  

    Бажаний вибір: GitHub Actions (простота налаштування для студентського репо, інтеграція з Docker, CI pipeline (lint → test → build)).  
15. Хостинг - забезпечує доступність сайту в інтернеті та надає серверні ресурси.  
    Варіанти:  
      - DigitalOcean (Droplet / App Platform)  
      - Heroku / Render (PaaS)  
      - AWS (EC2, RDS, S3)  
      - VDS / VPS  

    Бажаний вибір: DigitalOcean (Docker на Droplet або App Platform) (баланс ціни/зручності; простіше, ніж AWS; дає реальний VPS/об’єктне сховище та серверні налаштування).  
16. Логування та моніторинг - відстежує помилки, збирає статистику роботи системи та допомагає в аналізі продуктивності.  
    Варінати:  
      - Sentry (error tracking)  
      - Prometheus + Grafana (метрики)  
      - CloudWatch / Papertrail (логування)  

    Бажаний вибір: Sentry + базове логування в файли/console (Sentry швидко показує runtime-помилки і стек-трек).  
17. Панель управління - надає адміністраторам інтерфейс для керування товарами, замовленнями, користувачами і фінансами.  
    Варіанти:  
      - Вбудована Django Admin  
      - Панель управління на React (custom dashboard)  
      - Використання готових UI kit / shadcn, AdminLTE  

    Бажаний вибір: Django Admin (дає швидко готову адміністративну панель для управління товарами, замовленнями і платежами; економить час на реалізацію).  
18. Логістика - інтегрує зовнішні сервіси доставки, створює накладні та відстежує статус посилок.  
    Варіанти:  
      - Інтеграції з локальними API (Nova Poshta, Ukrposhta)  
      - Інтеграція з міжнародними перевізниками (DHL, UPS)  
      - Модуль manual-entry (ручне оформлення)  

    Бажаний вибір: Nova Poshta API (інтеграція демонстраційного потоку) (реалістична локальна інтеграція для України; доступна документація і sandbox-режими).  
19. Безпека - забезпечує захист даних і транзакцій, реалізує протоколи HTTPS, CSRF, XSS, перевірку вводу та управління секретами.  
    Варіанти:  
      - HTTPS (Let's Encrypt)  
      - CSRF, XSS-захист (фреймворк)  
      - Rate-limiting, input validation, secrets manager  

    Бажаний вибір: HTTPS + CSRF + валідовані ендпоінти + секрети в env (базові, але обов’язкові практики для захисту даних і платежів; демонструють знання безпеки у проєкті).
